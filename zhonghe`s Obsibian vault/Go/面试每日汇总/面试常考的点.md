### **系统设计类问题**

1. **简述你设计的银行系统的架构**
    
    - 问题要点：
        
        - 说明项目使用的设计模式、分层结构（如 MVC、Clean Architecture 等）。
            
        - 为什么选择这个架构？它如何帮助扩展、维护？
            
        - 如何组织各个模块：账户、转账、查询等？
            
2. **如何处理并发转账问题，避免并发冲突？**
    
    - 问题要点：
        
        - 数据库事务的使用（ACID 原则）。
            
        - 使用乐观锁或悲观锁的选择与实现。
            
        - 可能的死锁防范策略。
            
3. **如何设计银行系统的账户表结构？**
    
    - 问题要点：
        
        - 账户基本字段（如账号、账户类型、余额、状态等）。
            
        - 如何保证数据一致性和安全性？
            
        - 表与表之间的关联关系。
            
4. **你会如何设计一个支持大规模并发请求的银行系统？**
    
    - 问题要点：
        
        - 系统架构设计（例如微服务、水平扩展等）。
            
        - 数据库负载均衡、分片策略。
            
        - 异步处理、队列机制的使用。

### **编程与代码实现类问题**

1. **如何实现一个转账功能，涉及账户余额变更、日志记录、事务处理？**
    
    - 问题要点：
        
        - 如何确保资金转账的原子性？
            
        - 在数据库中如何使用事务？
            
        - 如何记录转账日志？
            
        - 代码实现，关注并发控制。
            
2. **如何实现用户的注册和登录功能，并加入 JWT 身份验证？**
    
    - 问题要点：
        
        - 用户注册时如何加密密码（如 bcrypt）？
            
        - 登录时如何生成 JWT Token？
            
        - Token 过期时间、刷新机制等安全策略。
            
3. **如何设计一个分页查询接口来获取账户交易记录？**
    
    - 问题要点：
        
        - 如何设计分页接口，接收 `page` 和 `size` 参数？
            
        - 数据库查询时如何处理大数据量的分页？
            
        - 代码实现与优化。

### **数据库设计类问题**

1. **在银行系统中如何设计事务表以及记录转账操作？**
    
    - 问题要点：
        
        - 事务表设计，包括字段如：交易ID、交易时间、交易金额、源账户、目标账户等。
            
        - 如何在系统中保持事务的完整性，避免数据丢失？
            
2. **如何优化银行系统的数据库查询性能，避免慢查询？**
    
    - 问题要点：
        
        - 如何通过索引优化查询性能？
            
        - 查询性能瓶颈的排查，如何使用 EXPLAIN 等工具分析。
            
        - 分库分表策略的应用。
            
3. **如何设计一套系统来防止 SQL 注入？**
    
    - 问题要点：
        
        - 使用预处理语句（Prepared Statement）。
            
        - 防范恶意输入，防止构造恶意 SQL。

### **安全性与身份验证类问题**

1. **如何防止银行系统中的账户密码泄露？**
    
    - 问题要点：
        
        - 密码加密： bcrypt、scrypt 等安全加密算法。
            
        - 账户登录时的安全措施（如验证码、二次验证等）。
            
        - 如何处理敏感数据的存储与传输？
            
2. **如何设计用户权限控制，防止权限滥用？**
    
    - 问题要点：
        
        - 如何区分不同权限级别的用户（管理员、普通用户等）？
            
        - 权限控制系统的设计与实现。
            
        - 最小权限原则和访问控制列表（ACL）。
            
3. **如何防止银行系统中的资金被恶意篡改或盗用？**
    
    - 问题要点：
        
        - 如何检测异常的资金交易（如金额过大、频繁转账等）？
            
        - 防止跨站请求伪造（CSRF）、跨站脚本攻击（XSS）等安全威胁。
            
        - 使用 HTTPS 加密通讯，防止数据被中间人窃取。

### **性能与优化类问题**

1. **如何优化银行系统中的查询性能？**
    
    - 问题要点：
        
        - 如何进行数据库优化，如索引、分区、查询优化。
            
        - 缓存机制的使用，例如 Redis 来缓存频繁访问的数据。
            
        - 异步处理机制，如何处理不需要实时响应的任务（如转账后的通知等）。
            
2. **在银行系统中如何实现高可用性和容错性？**
    
    - 问题要点：
        
        - 使用主从复制、分布式数据库等高可用架构。
            
        - 如何应对数据库宕机、网络故障等问题？
            
        - 如何设计一个灾备恢复方案？
            
3. **如何处理系统中大量交易数据的分析与统计？**
    
    - 问题要点：
        
        - 如何存储交易数据以便于后续分析？
            
        - 使用流式计算、批处理等方式进行实时分析。
            
        - 数据仓库、ETL（Extract-Transform-Load）等技术的使用。

### **测试与 CI/CD 类问题**

1. **如何为银行系统编写单元测试？**
    
    - 问题要点：
        
        - 如何为接口层（API）编写单元测试？
            
        - 使用哪些工具？如 `testing` 包、`testify` 等。
            
        - 如何进行数据库的模拟（Mocking）测试？
            
2. **如何使用 GitLab CI/CD 实现银行系统的自动化测试与部署？**
    
    - 问题要点：
        
        - CI/CD 流程：从代码提交到自动化测试、构建、部署的过程。
            
        - 如何在 CI 中自动执行单元测试、代码风格检查等？
            
        - Docker 容器化部署的流程。
